# Build-in-Public — social media journey

Separate from the project (lives in `research/social-media/`, never touches `studio/`).
Goal: post daily "what I built today" → grow connections + show I'm a strong dev.

## Files
- `posts.md` — ready-to-copy daily posts (newest first). X/Twitter + LinkedIn versions.
- `TEMPLATE.md` — fill-in template for a new day.

## Daily ritual (2 min)
1. Finish coding → read the latest entry in `studio/docs/DEVLOG.md`.
2. Copy `TEMPLATE.md`, write today's post at the TOP of `posts.md`.
3. Post the X version (thread) + the LinkedIn version. Done.

## Tips that grow a build-in-public account
- **Show, don't tell:** a screenshot / screen-recording > words.
- **One concrete thing per day** + a number ("8K export from a 720p laptop").
- **Share the struggle too** (bugs, rewrites) — people connect with honesty.
- **End with a hook/question** so people reply.
- Consistent hashtags: `#buildinpublic #indiehackers #reactjs #electron #webdev`
- Post the same to **X + LinkedIn** (LinkedIn = recruiters/clients, X = devs).
